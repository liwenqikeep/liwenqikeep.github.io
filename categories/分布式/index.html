分布式系统设计（三）：一致性算法 Paxos/Raft

分布式 分布式系统 Paxos Raft

前言 一致性算法是分布式系统的基石。它们解决的核心问题是：在多个节点之间，如何就某个值达成一致？ 即使其中部分节点故障或网络异常。 Paxos 和 Raft 是最著名的两个一致性算法。Paxos 更理论化，Raft 更工程化。

分布式系统设计（二）：分布式理论 CAP/BASE/一致性

分布式 分布式系统 CAP BASE

前言 分布式系统的核心挑战在于：网络不可靠、时钟不一致、节点可能故障。CAP 定理和 BASE 理论是指导我们在这些约束下做出设计取舍的基本原则。

分布式系统设计（一）：设计思想与方法论

分布式 分布式系统 设计思想 无状态

前言 构建分布式系统与构建单体应用的最大区别在于：你必须假定你所依赖的任何东西都可能失败。网络会断、机器会宕、磁盘会坏、消息会丢。 本文不讨论某个具体技术，而是总结分布式系统设计的核心思想和通用方法论——这些思想贯穿在所有分布式系统的设计和实现中。
